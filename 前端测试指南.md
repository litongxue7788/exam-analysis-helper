# 前端测试指南

## 🌐 访问地址
http://localhost:3000

## 🎯 测试目标
验证低置信度确认对话框在前端的显示效果

## 📋 测试步骤

### 1. 打开前端页面
在浏览器中访问: http://localhost:3000

### 2. 上传测试图片
- 点击上传按钮
- 选择刚才测试的4张图片：
  - test_image_1.jpg
  - test_image_2.jpg  
  - test_image_3.jpg
  - test_image_4.jpg

### 3. 观察分析过程
- 查看进度条显示
- 等待分析完成

### 4. 验证低置信度对话框
当系统检测到低置信度时，应该显示确认对话框：

**预期显示内容**:
- 标题: "识别置信度较低"
- 识别结果: "高二 数学"
- 置信度: "59%"
- 三个操作按钮:
  - "继续分析" (绿色)
  - "修正后继续" (蓝色) 
  - "取消分析" (灰色)

### 5. 测试用户交互
- 点击"修正后继续"按钮
- 验证年级/学科选择器是否出现
- 测试修正功能是否正常

## 🔍 关键验证点

### ✅ 应该看到的效果
1. **对话框显示**: LowConfidenceConfirmDialog组件正常渲染
2. **样式正确**: 使用LowConfidenceConfirmDialog.css样式
3. **数据正确**: 显示"高二 数学 59%"
4. **交互正常**: 三个按钮都可以点击

### ❌ 如果没有显示对话框
可能的原因：
1. 前端组件未正确集成到Report.tsx
2. 后端数据格式不匹配
3. 前端状态管理有问题

## 🛠️ 调试方法

### 1. 检查浏览器控制台
- 按F12打开开发者工具
- 查看Console是否有错误信息
- 检查Network请求是否正常

### 2. 检查组件状态
在Report.tsx中添加调试日志：
```javascript
console.log('Analysis result:', result);
console.log('Should show dialog:', result?.recognitionInfo?.confidence?.level === 'low');
```

### 3. 验证后端数据
确认后端返回的数据包含：
```json
{
  "recognitionInfo": {
    "confidence": {
      "level": "low",
      "score": 0.59
    },
    "warnings": ["⚠️ 识别置信度较低，建议人工确认"]
  }
}
```

## 📱 移动端测试
如果需要测试移动端效果：
- 在浏览器中按F12
- 点击设备模拟器图标
- 选择手机设备进行测试

## 🎉 测试完成标准
- ✅ 对话框正常显示
- ✅ 数据显示正确
- ✅ 用户交互正常
- ✅ 样式美观合理
- ✅ 移动端适配良好
