# 用户使用说明 - 访问口令配置

## 首次使用配置

当你第一次访问试卷分析助手时，可能会看到"口令失效"或"访问口令错误"的提示。这是正常的安全机制。

### 配置步骤

#### 1. 打开设置界面

在手机或电脑浏览器中：
- 访问试卷分析助手网站
- 点击右上角的**设置图标**（⚙️齿轮图标）

#### 2. 输入访问口令

在设置界面中：
- 找到"试点访问口令"输入框
- 输入管理员提供的访问口令
- 点击"保存"按钮

#### 3. 刷新页面

- 关闭设置界面
- 刷新页面（下拉刷新或重新打开）
- 现在应该可以正常使用了

## 获取访问口令

请联系系统管理员获取访问口令。

管理员可以通过以下方式查看：
```bash
# SSH登录服务器
ssh root@服务器IP

# 查看口令
cd /opt/exam-analysis-helper
bash 查看访问口令.sh
```

## 常见问题

### Q1: 为什么需要访问口令？

A: 访问口令用于：
- 控制试点范围
- 防止服务滥用
- 管理使用配额
- 保护服务器资源

### Q2: 口令保存在哪里？

A: 口令保存在浏览器的localStorage中，即使关闭浏览器也会保留。

### Q3: 更换设备后需要重新输入吗？

A: 是的。每个设备/浏览器都需要单独配置口令。

### Q4: 忘记口令怎么办？

A: 联系管理员重新获取访问口令。

### Q5: 可以分享口令给别人吗？

A: 可以，但请注意：
- 每个口令有使用配额限制
- 多人共用可能导致配额用尽
- 建议每人使用不同的口令

### Q6: 输入口令后还是提示错误？

A: 请检查：
1. 口令是否输入正确（注意大小写）
2. 是否有多余的空格
3. 网络连接是否正常
4. 尝试清除浏览器缓存后重试

### Q7: 如何清除已保存的口令？

A: 在设置界面：
- 清空"试点访问口令"输入框
- 点击保存

或在浏览器控制台执行：
```javascript
localStorage.removeItem('trialAccessCode');
```

## 配额限制

当前配置的使用限制：
- 每分钟请求数：30次/口令
- 每日请求数：350次/口令

如果超过限制，会收到"请求过于频繁"的提示，请稍后再试。
