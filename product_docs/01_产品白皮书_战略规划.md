# 试卷分析助手：产品演化与迭代计划（战略版）

> **面向对象**：核心产研团队、教研专家  
> **当前阶段**：MVP 验证期（聚焦毕业班单科闭环）  
> **核心使命**：把一张试卷的分数，转化为一套可执行、可验收的提分计划。

---

## 负一、开发约束宣言（红线）

> **所有开发与迭代必须遵守以下约束，防止需求蔓延与方向跑偏。**

1.  **闭环优先原则**：任何功能如果没有对应的“训练”或“验收”环节，**坚决不做**。不生产无法闭环的“信息垃圾”。
2.  **不扩散原则**：在 MVP 阶段（阶段 1），严禁扩展到非毕业班年级、严禁做社区/社交功能、严禁做大而全的内容库。
3.  **合规底线**：严禁提供“直接抄袭答案”的路径；所有输出必须以“引导思考”和“训练掌握”为目的。
4.  **数据最小化**：不收集非必要的隐私数据；试卷原始图片处理后即刻脱敏，不长期存储未授权的个人信息。

## 零点五、四观共识（我们的产品方法论）

### 1. 学习本质观：洞见时刻驱动的认知结构改变
- 学习不是信息的搬运，而是认知结构的改变与心流状态的获得；有效的标志是持续的“洞见时刻”。
- 产品输出必须优先制造“洞见时刻”：把错误拆到可解释的错因，把改法拆到可执行的动作，把进步拆到可验证的证据。

### 2. 产品定位观：我们设计的是状态体验与心流环境
- 我们不是建造“学习工具”，而是设计“状态体验”。用户进入产品，应当更像进入一个可专注的心流环境。
- 关键设计标准：少干扰、低摩擦、少而精、节奏明确（下一步永远清楚）。

### 3. 用户关系观：用户是驾驶舱内的飞行员
- 用户不是被动接受者，而是“驾驶舱内的飞行员”。产品提供掌控感，并作为认知伙伴提供支持。
- 关键设计标准：所有结论可追溯、可调整、可选择；用户能看到“为什么给我这个建议”与“我如何确认它有效”。

### 4. 价值提供观：确定性与内化效率
- 我们提供的核心价值不是知识本身，而是“确定性”（薄弱点确定、路径确定、进步幅度确定）与“内化效率”。
- 关键交付物是可执行、可验收的闭环，不是静态报告：复盘卡（确定薄弱点）→训练卡（确定路径）→验收卡（确定进步）。

## 零、核心价值与北极星指标

### 1. 一句话定位
我们不做“拍照搜题给答案”的工具，我们做**“考试复盘与提分闭环”**的智能助手。  
用户交付一张试卷，我们交付一套**“三卡闭环”**（复盘卡、训练卡、验收卡）。

### 2. 北极星指标（North Star Metric）
- **核心指标**：**“有效闭环率”**  
  定义：用户生成复盘卡后，在 48 小时内完成训练并进行验收测试的比例。
- **过程指标**：
  - 错因复发率（同类错因在下一次考试中的减少比例）
  - 训练完成率（训练卡任务的执行比例）
  - 验收通过率（验收题目的正确率）

### 3. 不做清单（边界控制）
- **不做**单纯的“拍照秒出解析”（避免同质化竞争，避免惰化思维）。
- **不做**大而全的内容/课程/社区平台（专注工具属性，内容可接入第三方）。
- **不做**无法闭环的“泛泛建议”（所有建议必须对应具体的训练任务）。

## 一、目标人群与痛点分层

### 1. 切入点：初/高三毕业班学生
- **特征**：目标明确（中考/高考）、时间极度稀缺、任务无边界、焦虑感强。
- **痛点**：
  - 刷了很多题，分数不涨（无效努力）。
  - 知道错了，但不知道怎么彻底改（缺乏方法）。
  - 考场发挥不稳定，非智力因素丢分多（审题/计算/心态）。
- **交付重点**：**稳定提分**（保底分+易错点清零）。

### 2. 拓展点：考公/职业职称人群
- **特征**：自律难、时间碎片化、主观题评分标准模糊。
- **痛点**：
  - 申论/案例分析不知道怎么拿分（评分点对齐难）。
  - 没人监督，复习进度不可控。
  - 面试/口试缺乏模拟环境。
- **交付重点**：**评分对齐**（主观题客观化）+ **强制闭环**（碎片时间管理）。

## 二、产品演化路径（Roadmap）

### 阶段 1：验证期——毕业班单科闭环（当前重点）
**目标**：在数学或英语单科上，跑通“试卷 -> 三卡 -> 提分”的最小闭环，验证核心价值。

- **功能模块**：
  - **复盘卡（诊断）**：
    - 错因强制归类：知识漏洞、方法错误、审题失误、计算失误、表达规范、策略失误。
    - 输出：每类错因对应 1 条“最短改法”（如：审题圈画关键词、计算分步草稿）。
  - **训练卡（行动）**：
    - 生成“今日任务”：15-30 分钟/组。
    - 分层推题：基础巩固（保底）-> 变式训练（纠偏）-> 迁移提升（冲刺）。
  - **验收卡（闭环）**：
    - 生成 3-5 道同型小测题。
    - 判定“掌握度”：通过则翻篇，不通过则触发二次训练。

- **关键动作**：
  - 采集最近 3-5 份试卷数据，建立初始“薄弱点地图”。
  - 建立“错因-训练策略”映射库（教研重点）。

### 阶段 2：成长期——全科覆盖与考场策略
**目标**：从单科扩展到全科，引入非智力因素干预，形成完整的“备考教练”。

- **新增功能**：
  - **全科看板**：多学科错因分布，识别跨学科共性问题（如：审题习惯差）。
  - **考场策略包**：
    - 基于历史数据，生成个性化“答题时间表”与“检查清单”。
    - 考前 1 小时提醒：重点看哪些易错点。
  - **教师/家长端联动**：
    - 家长端：只看“监督清单”（今天孩子要练哪 3 组题，完成没有）。
    - 教师端：班级共性错因汇总，自动生成讲评课件素材。
  - **私人助理升级**：支持对话式复盘（“这道题我当时是怎么想的...”），引导学生自我发现逻辑漏洞。

### 阶段 3：拓展期——人群破圈与生态构建
**目标**：将“闭环方法论”迁移至考公/职称领域，构建内容生态。

- **新增场景**：
  - **主观题评分对齐**（考公申论/职称案例）：
    - 用户输入答案，AI 按评分点逐条打分，并给出“升格示例”。
    - 核心价值：让主观题复习变得像客观题一样可量化。
  - **面试模拟教练**：
    - 语音交互，模拟真实追问。
    - 输出：回答逻辑复盘 + 表达体态建议。
  - **开放内容生态**：
    - 允许名师/机构入驻，提供针对特定考试的“训练卡”内容包。

## 三、MVP 核心功能规格（阶段 1）

### 1. 输入层
- **支持格式**：图片（试卷/答题卡）、PDF、手动录入分数/错题号。
- **必要元数据**：考试类型（中考/高考/模拟）、学科、满分值。

### 2. 处理层（AI Agent）
- **错因分析引擎**：
  - 基于题目解析与用户作答（或错题描述），推理最可能的错因类别。
  - *技术难点*：如何区分“真的不会”和“粗心”？（策略：基于历史数据与用户确认交互）。
- **任务生成引擎**：
  - 规则：优先级排序（高频错因 > 偶然失误），总量控制（每日 < 45分钟）。

### 3. 输出层（用户界面）
- **复盘报告**：
  - **不展示**：密密麻麻的知识点网状图（太吓人）。
  - **展示**：3 个核心待办项（Top 3 Weakness）。
- **打印/导出**：
  - 支持将“训练卡”和“验收卡”导出为 PDF，方便打印练习（还原纸笔考试习惯）。

## 四、技术与教研协同

- **技术侧**：
  - 重点打磨 Prompt 工程，确保错因分析的准确性与“人味”。
  - 构建“题型-错因-训练”的图谱结构数据库。
- **教研侧**：
  - 定义各学科的“标准错因集”与“改法SOP”。
  - 生产/筛选高质量的“同型题库”用于训练与验收。

## 五、风险与对策

| 风险点 | 描述 | 对策 |
| :--- | :--- | :--- |
| **幻觉风险** | AI 对题目解析错误或生成错误答案 | 引入“置信度”评分，低置信度转人工或提示用户；优先使用经过校验的题库。 |
| **用户惰性** | 拿到训练卡但不执行 | 引入“打卡激励”；家长端强监督；任务颗粒度切分得更小（5分钟/组）。 |
| **数据隐私** | 试卷包含个人信息 | 客户端本地脱敏/打码；服务器不留存原始卷面，只存结构化数据。 |

## 六、市场展望与核心竞争力 (Market & Why Us)

### 1. 市场预调研与机会点
- **红海现状**：“拍照搜题”类工具（如作业帮/小猿）已占领“找答案”的心智，但因“惰化思维”被监管限制，且家长/老师普遍反感。
- **蓝海缺口**：
  - **家长侧**：需要一个“能代替我监督孩子复习”的专业管家，而不只是一个题库。
  - **学校侧**：需要一个“能真正落实分层作业”的教研助手，而不只是阅卷工具。
  - **学生侧**：毕业班学生急需“在最短时间内涨分”的策略工具，痛恨无效刷题。
- **我们的机会**：做**“反内卷的提分工具”**——用最少的题（精准诊断+变式训练）换最大的分。

### 2. 核心优势：从“给结果”到“管过程”
- **别人做的是**：“拍照 -> 识别 -> 题库匹配 -> **给答案解析**”。
  - *结果*：学生抄答案，家长看不懂，老师不知道学生真懂假懂。
- **我们做的是**：“试卷 -> 诊断 -> 任务卡 -> **强制验收**”。
  - *结果*：学生必须做题验证，家长只看完成率，老师看到真实掌握度。

### 3. 差异化护城河
- **输出形态差异**：竞品交付“报告”，我们交付“三卡”（复盘/训练/验收）。我们的产品是**动词**（让用户动起来），竞品是**名词**（给一个静态文档）。
- **数据价值差异**：竞品积累的是“搜题记录”（杂乱无章），我们积累的是“错因消亡史”（高价值的个人成长数据）。
- **合规性壁垒**：完全避开“惰化思维”的监管雷区，反而是“促进深度思考”的合规典范，更容易进入校内或被家长接受。

### 3. 市场立足点
- **极度功利但有效的切口**：毕业班学生没时间做无效题，我们的“少而精 + 必考点”策略最打动他们。
- **降维打击**：用教研级的“诊断闭环”去打市面上泛滥的“技术流搜题”，对家长和老师的吸引力是碾压级的。

## 七、关键问题价值排序与优化方向（面向产品开发）

本节用于将“家长/学生的关键问题”转化为产品决策优先级，避免讨论热闹、开发跑偏。

### 1. 价值排序（从高到低）

S 级（决定是否成立）：
- 家长是否愿意付费，以及唯一拒付原因是什么
- 如何证明有效：进步证据如何展示，未进步如何解释
- 如何防止变相抄答案：提示链与验收机制如何设计
- 孩子不自律怎么办：闭环如何在低家长参与下仍能跑起来
- 隐私与信任：卷面与成绩如何最小化处理与可控留存

A 级（决定是否易用与可扩散）：
- 家长最想减少哪一种痛苦（时间/冲突/焦虑/无效努力）
- 家长愿意参与到哪一步就足够（参与上限与默认模式）
- 产品形态：学生端/家长端分别长什么样，是否能一屏进入心流与闭环
- 输出形态：报告/清单/任务/打印件，哪一种更能进入心流
- 替代方案对比：用户现在用什么，为什么不满意

B 级（决定品牌叙事与长期潜力）：
- “学习驾驶舱/心流环境”如何讲清楚、如何在界面落地
- 我们对学习的认知是什么（对外表达版本）

### 2. 与当前方案对照（已覆盖与缺口）

已覆盖：
- 三卡闭环作为核心交付物（复盘卡→训练卡→验收卡）
- 北极星指标为“有效闭环率”（从生成到训练+验收）
- 开发红线（不扩散、合规、数据最小化）

明显缺口（需要补齐为“确定性”产品化机制）：
- 有效性证据的标准化呈现：家长一眼能看懂的“进步证据面板”
- 错因可信度与可回溯：结论依据、置信度、用户确认与纠偏机制
- 反抄答案策略的具体交互：提示链分层、解锁条件、验收闸门
- 低参与闭环：家长不盯也能跑的默认策略与失败时的兜底
- 付费与包装：家长愿意为哪一种确定性付费（按周/按月/按学期）
- 产品形态收敛：学生驾驶舱与家长仪表盘的默认布局与关键按钮

### 3. 优化方向（按优先级给出可执行动作）

P0（必须先做，才能谈增长）：
- 建立“证据链”输出：每个错因都给出来源（题型/得分/历史复发）+ 置信度
- 建立“验收判定”规则：通过/不通过的可解释逻辑与二次训练触发
- 设计“最小提示链”：只给下一步提示，不给完整抄袭路径，且必须验收
- 设计“家长驾驶舱”：只展示完成率、验收结果、错因复发变化与明日任务
- 固化“MVP界面骨架”：所有页面围绕三卡闭环与证据链，避免信息堆砌

P1（提升留存与体验）：
- 上手即闭环：首次使用 3 分钟内进入训练，不先看长报告
- 任务颗粒度可控：默认 15–30 分钟/天，允许用户调节但不破坏验收
- 失败解释机制：分数没涨时，告诉家长“是哪一类错因未消亡/验收未通过”

P2（形成壁垒）：
- 错因-改法SOP教研共建：把“方法增长”固化为可复用规则库
- 跨学科共性问题识别：把“审题/计算/表达”等迁移能力做成产品亮点

### 4. 产品形态（MVP 形态收敛）

本节用于约束界面与交互，防止“功能越加越多、体验越来越重”。

学生端（学习驾驶舱）：
- 一屏只包含三块：复盘卡（薄弱点与证据）/训练卡（今日任务）/验收卡（通过判定）
- 每次只抓 1–2 个优先级目标，默认 15–30 分钟可完成
- 允许选择与调整，但不破坏闭环：可换顺序、可调难度、不可跳过验收

家长端（监督仪表盘）：
- 一屏只回答四个问题：今天做没做、验收过没过、错因复发降没降、明天做什么
- 不展示长解析与题目细节，只展示证据与趋势，降低陪学冲突

打印/导出（纸笔友好）：
- 训练卡与验收卡支持导出成一页纸，适配纸笔练习习惯

对话入口（认知伙伴，而非聊天机器人）：
- 对话只做三件事：解释证据、帮助做选择、引导复盘与验收

## 八、P0级任务验收条款 (Acceptance Criteria for P0 Tasks)

> **本节是 MVP 开发与测试的“红线标准”，任何一条不满足，均视为未交付。**

### 1. “证据链”输出验收条款

- **AC-1.1 (错因来源可追溯)**：学生驾驶舱中展示的每一个“错因”，必须明确标注其判定的**核心依据**。依据至少包含：【题号】、【原始得分/满分】、【关联知识点/题型】。
- **AC-1.2 (历史复发可量化)**：对于重复出现的错因，必须在旁边用数字或标签清晰展示**历史复发次数**（例如：“审题失误 (复发3次)”）。
- **AC_1.3 (置信度必须展示)**：每一个由 AI 分析得出的错因，必须附带一个**置信度评级**（高/中/低）。低置信度的结论必须有引导用户确认或纠错的入口。
- **AC-1.4 (无证据不展示)**：如果系统无法为某个错因找到明确的试卷证据，则**不允许**在复盘卡中展示该错因。

### 2. “验收判定”规则验收条款

- **AC-2.1 (通过/不通过逻辑明确)**：验收卡的“通过”标准必须是**具体且可解释的**。例如：“3题全对”或“答对2题且核心步骤分拿到”。此规则必须在验收开始前对用户可见。
- **AC-2.2 (二次训练自动触发)**：如果验收结果为“不通过”，系统必须**自动触发**二次训练流程。二次训练可以是“原题重做”、“同类题减量训练”或“查看最小提示后重做”，但不可直接跳过。
- **AC-2.3 (验收结果不可手动修改)**：用户或家长**不能**手动修改“通过/不通过”的最终判定结果，确保数据的客观性。

### 3. “最小提示链”设计验收条款

- **AC-3.1 (严禁直接给答案)**：在训练或二次训练过程中，提示功能**严禁**一次性展示完整的解题步骤或最终答案。
- **AC-3.2 (提示必须分层)**：提示必须至少分为**三层**：① 引导审题（“请注意题目中的‘至少’二字”）；② 提示思路（“可以尝试构造辅助线”）；③ 给出关键步骤的起始公式。每层提示需要用户主动点击才能解锁。
- **AC-3.3 (使用提示必须验收)**：任何查看了“关键步骤”级别提示的题目，在本次训练结束后，其同类题必须在下一次或本次验收卡中**强制出现**，以验证是否真正掌握。

### 4. “家长驾驶舱”验收条款

- **AC-4.1 (一屏四要素)**：家长端默认首页（仪表盘）**只允许**展示四个核心模块：【今日任务完成率】、【最近验收通过率】、【高频错因复发趋势图】、【明日核心任务预告】。
- **AC-4.2 (严禁展示题目细节)**：家长仪表盘**严禁**默认展示任何具体的题目、解析、或学生的手写答案，避免引发直接的学术辅导冲突。如需查看，必须经过二次点击或授权。
- **AC-4.3 (趋势优先于单点)**：所有数据展示（如错因复发）必须优先使用**趋势图**（例如：折线图、柱状图），而非罗列单次数据，帮助家长建立过程改进的体感。

### 5. “MVP界面骨架”验收条款

- **AC-5.1 (学生端一屏三卡)**：学生端主界面（学习驾驶舱）必须在**一屏之内**清晰展示“复盘卡”、“训练卡”、“验收卡”三个区域的入口或核心状态，无需滚动或跳转。
- **AC-5.2 (三分钟上手原则)**：新用户首次登录，跟随引导必须在**三分钟内**能够开启第一次“训练卡”任务，不允许被复杂的设置或冗长的报告所阻碍。
- **AC-5.3 (关键操作按钮恒在)**：【开始训练】和【开始验收】按钮必须是界面中最醒目的**高优先级**操作按钮。

## 九、可执行版本定义（V0.1–V0.3）

> 本节用于把“愿景”收敛为可执行的版本与验收口径，避免毕业班场景需求发散。

### 1. 本轮要解决的核心矛盾

- 毕业班学生时间紧、任务重，容易陷入“题海战”并产生挫败感。
- 市面同类工具多交付“解析/答案”，但无法解决“知道错了 → 怎么改 → 是否真的掌握”的闭环问题。
- 我们的交付必须从“报告”升级为“闭环结果”：用最少的训练，换最大、可验证的分数提升体感。

### 2. 目标用户（V0.x）

- 学生：初三、高三（默认 15–30 分钟/天的闭环节奏）。
- 老师：毕业班数学/语文/英语老师（关注分层作业、讲评效率、口径可控）。
- 家长：低参与监督（只看趋势与验收结果，不介入题目细节）。

### 3. 核心闭环（V0.1 必须跑通）

- 复盘：把学生“已做题/作业/试卷图片”转为【错因 + 证据 + 置信度 + 最短改法】。
- 训练：每个错因生成 1 组可执行任务（基础巩固 → 变式训练 → 迁移提升），默认 15–30 分钟内完成。
- 验收：对本次训练对应错因生成 3–5 题小测或二次提交任务，明确通过规则；不通过触发二次训练。
- 沉淀：形成“错因消亡史”（错因复发次数下降、验收通过率上升），用于对学生与家长解释“为何有效/为何暂时无效”。

### 4. 学科场景收敛（V0.1 范围）

数学（解答题/应用题）：
- 评价重点：审题条件是否完整、建模是否正确、关键等式/关键步骤是否成立、步骤分是否丢失、计算/变形是否出错。
- 输出必须可执行：指出从哪一步开始跑偏，并给最短改法（例如“设元+列式模板”“检验单位/范围”）。

语文（阅读与作文）：
- 阅读：按题型给“得分点清单 + 原文依据定位 + 缺失项”，避免泛泛讲道理。
- 作文：按评分维度给“扣分点 + 证据句 + 单点改写任务”，默认不生成可直接抄的终稿。

英语（阅读与写作）：
- 阅读：给“定位证据句 + 错因（细节/主旨/推断/词义）+ 同题型微练”。
- 写作：给“任务完成度（要点覆盖）+ 结构 + 语言错误分型（时态/搭配/从句等）+ 可替换表达包”，以二次改写验收。

### 5. V0.1 关键功能清单（必须/不做）

必须（MVP）：
- 单题/单页拍照提交，返回结构化结果：【是否正确/得分】、【错因标签】、【证据】、【置信度】、【最短改法】。
- 生成“训练卡”与“验收卡”，并可打印/导出为一页纸，适配纸笔训练。
- 最小提示链：提示分层解锁，默认不直接给完整解法/终稿；使用高层提示必须强制进入验收。
- 失败可解释：判定低置信度时，不强行下结论，必须给出“需要补拍/需要老师确认”的路径。

不做（V0.1 明确排除）：
- 整卷自动切题与题号定位（后续版本再做）。
- 大而全题库与课程内容（只做“针对性小题组/小测”）。
- 无验收的报告堆砌与泛泛建议。

### 6. V0.1 验收指标（试点必须采集）

- 首次闭环完成时间：首次使用 3 分钟内进入训练；48 小时内完成训练 + 验收的比例（有效闭环率）。
- 单次任务时长：默认 15–30 分钟完成；超时比例与原因。
- 错因复发率：同错因在下一次验收/考试中出现的下降比例。
- 老师节省时间感知：批改/统计/布置分层作业的时间是否减少（以访谈量化区间记录）。

### 7. V0.2–V0.3 里程碑（在 V0.1 验证后推进）

V0.2（教师端闭环）：
- 班级共性错因分布与分层作业一键生成。
- 教师可控口径：可改判、可补评分点、可修正错因标签并沉淀为本班口径。

V0.3（家长端闭环）：
- 家长驾驶舱一屏四要素：完成率、验收通过率、错因复发趋势、明日任务。
- 失败解释机制：分数未涨时，明确是哪类错因未消亡/哪次验收未通过。

## 十、首页（上传页）最终方案：专业“上传指挥台”（已定稿）

用户在首页只做三件事：**上传 → 校验 → 开始分析**。

- **A 区：任务状态条（置顶）**
  - 文案示例：`已上传 6 张｜预计 25 秒｜页序可拖动调整｜缺页会提示`
  - 所有不满足条件（未上传/张数不足/格式不对/缺页）都在这里提示，并同步禁用主 CTA

- **B 区：队列表格（核心舞台）**
  - 行：每一张图片/一个文件
  - 列（第一期必须有）：缩略图 / 名称 / 来源（拍照/相册/Excel） / 页码 / 状态 / 操作
  - 交互（第一期必须有）：拖拽排序（决定页序）、行内预览、删除、旋转

- **C 区：主 CTA 与近邻操作（右下角聚合）**
  - 主按钮：`开始分析`
  - 近邻按钮：`继续添加`、`清空队列`
  - 原则：相近相关联的按钮固定放在同一位置，避免四处找入口

## 十一、信息架构最终形态：全站统一为“控制台三段式”

- **顶部：全局工具条**（返回/设置/导出/帮助）
- **中部：主舞台**（当前页核心任务：上传队列 / 三卡闭环 / 练习与验收）
- **右侧/底部：指挥官面板**（进度、任务清单、关键 CTA、即时反馈）
