教师试用与闭环验收指南（V0.1）

适用对象：项目发起人 / 管理者  
测试时间：首次小范围教师试用（建议 60–90 分钟）  
测试范围：毕业班（初三/高三）数学 + 语文 + 英语（以实际准备材料为准）

---

零、快速检查清单（5 分钟版本）

- [ ] 后端终端正在运行：`npx ts-node server.ts`，无报错。  
- [ ] 前端终端正在运行：`npm run dev`，能看到 `Local / Network` 地址。  
- [ ] 管理员密码已配置为 `ADMIN_PASSWORD=（自行设置的强密码）`，后端已重启。  
- [ ] 至少准备好 1 套数学 + 1 套语文 + 1 套英语试卷（照片或成绩表）。  
- [ ] 确认老师设备连在与你同一个 Wi‑Fi/局域网。  
- [ ] 在老师群里准备好访问链接（如 `http://192.168.31.42:3000/`）和简短说明。

---

一、测试目标

1. 体验目标
   - 老师是否能在简单说明下独立走完一次完整流程：
     - 打开链接 → 填写学生与考试信息 → 拍照提交（单题/单页/作文/阅读） → 生成结果 → 生成练习/验收。
   - 界面是否清晰，步骤是否自然（无需太多口头解释）。
2. 教学价值目标
   - 结果是否能直接用于“讲评 + 分层训练 + 复测验收”：
     - 错因是否清晰、可追溯（有证据，不是凭空结论）。
     - 建议是否可执行（能落到具体动作与任务量）。
     - 练习/验收是否能覆盖识别出的薄弱点（同型 + 变式 + 迁移）。
   - 主观题（作文/阅读/英语写作/数学解答题步骤分）口径是否可接受，是否需要“老师可改判”的强入口。
3. 沟通与助理目标（如已启用私人助理）
   - 老师是否愿意用对话方式快速解读报告、生成任务卡或整理讲评要点。
4. 管理与推广目标
   - 验证“管理员配置一次大模型秘钥，老师直接使用”的流程。
   - 收集反馈，用于后续版本的功能优先级规划。
   - 采集“家长是否愿意买单”的真实阻力点：最担心什么、需要什么证据才付费。

二、测试前准备（管理者）

1. 后端秘钥配置
   - 确保已在 `backend/.env` 中配置：
     - 大模型服务商所需的 `API_KEY` 和 `MODEL_ID`。
     - `ADMIN_PASSWORD`（管理员密码，自行设定，明天只由你本人使用）。
   - 已实现的管理界面允许你在前端设置里通过“管理员配置”按钮，将 Key/Model 保存到服务器。

2. 启动服务
   - 后端（命令行在 `D:\试卷分析助手\backend`）：
     - `npx ts-node server.ts`
   - 前端（命令行在 `D:\试卷分析助手\frontend\web`）：
     - `npm run dev`
   - 确认终端输出中有本机和局域网访问地址，例如：
     - `Local:   http://localhost:3000/`
     - `Network: http://192.168.xx.xx:3000/`

3. 网络环境确认
   - 自己的电脑和老师的设备必须在同一个 Wi‑Fi/局域网下。
   - 建议提前记下可用的局域网访问地址，例如：
     - `http://192.168.31.42:3000/`

4. 准备测试试卷/数据
   - 本次以 V0.1 的“单题/单页闭环”为主，建议每科准备 2–3 份材料（照片即可）：
     - 数学：1 道应用题/解答题（最好含步骤分）+ 1 道易错小题（计算/审题类）。
     - 语文：1 道课外阅读主观题（含评分点更佳）+ 1 篇作文（或作文片段）。
     - 英语：1 道阅读理解题（最好包含推断/主旨类）+ 1 篇写作（或写作片段）。
   - 可选加分材料（若老师方便提供）：
     - 标准答案/评分点（能显著提升主观题可靠性）。
     - 老师自己的判分结果（用于对齐口径）。

5. 准备用于老师邀请的文字
   - 提前在老师微信群中准备一段简短说明（如何打开链接、主要用途）。
   - 示例话术可参考之前生成的“老师使用超简版说明”。

三、测试流程（现场运行顺序）

步骤 0：对齐本次测试边界（30 秒）

1. 本次重点验证“闭环是否成立”，不要求整卷自动切题、不要求覆盖所有题型。
2. 判题结果必须带证据与置信度；低置信度允许提示补拍或老师确认。

步骤 1：管理者自测与演示

1. 自己先用本机浏览器打开：
   - `http://localhost:3000/`
2. 打开右上角设置：
   - 填写学生信息（可以用虚拟学生名）。
   - 学科设为“数学”，填写考试名称、时间。
3. 使用数学材料进行一次完整演示（单题/单页即可）：
   - 使用“拍照录入”上传 1–2 张图片。
   - 点击“生成个人分析报告”。
   - 展示本次闭环必须出现的内容：
     - 错因结论是否有证据、是否有置信度提示。
     - 建议是否可执行（具体动作 + 任务量）。
     - “巩固练习/练习卷”是否可直接打印或保存。
     - 是否存在变相给答案的路径（提示链是否需要主动解锁）。

步骤 2：管理者配置大模型（如当天还未配置）

1. 仍在本机前端页面，点击右上角设置。
2. 在设置弹窗底部点击“管理员配置”按钮。
3. 在管理员区域中：
   - 选择模型服务商（如“豆包/通义/智谱”）。
   - 输入管理员密码（与 `.env` 中 `ADMIN_PASSWORD` 一致）。
   - 填写模型 ID/接入点 ID。
   - 填写 API Key。
   - 点击“保存大模型配置到服务器”，等待成功提示。
4. 成功后，即使刷新页面或老师从其他设备访问，也会使用同一套配置。

步骤 3：邀请老师加入测试

1. 在老师群里发送访问链接：
   - 明确说明需要在同一 Wi‑Fi 下访问，例如：
     - `http://192.168.31.42:3000/`
2. 附上极简使用说明（可复制）：
   - 打开链接 → 填学生与考试信息 → 上传试卷/成绩 → 点“生成个人分析报告”。
3. 强调测试目的：
   - 看看整体流程是否顺畅。
   - 看报告中的表达是否贴合学科教学实际。

步骤 4：老师自行试用

建议给每位老师一个具体“小任务”，方便引导：

- 数学老师：
  - 用 1 道应用题/解答题完成一次闭环：
    - 拍照提交 → 查看错因与证据 → 打开练习卷 → 判断同型题是否对准问题。
  - 关注点：
    - 是否能定位到“从哪一步开始错”，而不是只说知识点名字。
    - 练习是否能覆盖：审题/建模/步骤分/计算四类典型丢分点。

- 语文老师：
  - 用 1 道阅读主观题 + 1 篇作文片段完成两次闭环（可分两次上传）：
    - 阅读：得分点是否被拆清楚，是否能定位原文依据。
    - 作文：扣分点是否具体，是否能给“单点改写任务”（而非直接给可抄终稿）。

- 英语老师：
  - 用 1 道阅读题 + 1 段写作完成两次闭环：
    - 阅读：是否给出证据句与错因（主旨/细节/推断/词义）。
    - 写作：是否能把错误分型（时态/搭配/从句等）并给出可替换表达包。

四、测试中的观察要点（你需要记录的内容）

1. 使用难度相关
   - 老师是否能在 1 分钟之内理解首页布局（“这是干什么的”）。
   - 老师是否能在少量文字说明下，自己完成一次分析。
   - 哪一步最容易卡住（例如：上传文件、学科选择、生成报告等待时间）。

2. 报告内容相关
   - 老师最喜欢报告中的哪一个模块（如：题型与能力分析、主要问题、建议）。
   - 有哪些提示语或用词看起来“不专业”或“不符合本学科口语习惯”。
   - 对家长可读性：老师觉得“如果给家长看”，哪些地方需要简化或调整。
   - 主观题可信度：哪些题型最容易误判，误判时是否能被“证据/置信度/老师确认”机制兜住。
   - 合规性：是否存在一键获得完整解法/作文终稿的路径（如有，必须记录具体入口与复现步骤）。

3. 性能与稳定性
   - 生成一份报告大概耗时多少秒。
   - 是否出现错误提示（如“分析失败”等），出现时的场景和操作。
   - 同一时间多位老师访问时，系统是否卡顿。
4. 私人助理（如已启用）
   - 回复是否贴合学科语气与老师口吻，是否能直接用于讲评/沟通。
   - 是否能稳定输出“下一步动作/任务卡/复测建议”，而不是泛泛而谈。

五、测试后收集反馈（建议用简单表格或问卷）

建议按老师学科整理反馈，可先口头记录，之后再汇总到表格。

推荐问题：

1. 流程体验
   - 第一次使用时是否需要他人指导？
   - 哪一步让你最犹豫或不确定？

2. 教学价值
   - 报告中对你最有价值的 1–2 个模块是哪些？（可多选）
   - 你希望报告中增加什么内容？（如：知识点分布图、班级横向对比等）

3. 内容表达
   - 请举出 1–2 句你觉得需要修改的“能力描述/建议话术”，并给出建议修改方向。

4. 给家长使用的可行性
   - 如果未来开放给家长，你觉得需要增加或弱化哪些内容？
   - 家长愿意为哪种“确定性”买单：省时间/少冲突/看得见的进步证据？
   - 家长唯一的拒付原因是什么：不信任、担心抄答案、看不懂、没效果、隐私？

5. 其他建议
   - 例如：导出格式、手机端显示效果、数据导入方式（和现有 Excel/系统对接）等。
6. 私人助理（如已启用）
   - 你最希望助理帮你完成哪 1 件事？（报告解读/任务卡/讲评要点/分层作业/家校话术）

六、测试结束后的下一步行动建议

1. 综合整理反馈
   - 按学科归类关键意见（数学/语文/英语）。
   - 找出：
     - 高频提到的问题。
     - 各学科共同关注的改进点。

2. 形成下一版本优先级
   - 优先考虑：
     - 影响“是否愿意继续用”的问题（流程卡点、体验问题）。
     - 老师一致认为“很有帮助”的模块，进一步增强。
   - 将“语文/英语能力描述优化”、“家长版简化报告”等列入后续迭代计划。

3. 对外沟通
   - 在老师群里简单回访：
     - 感谢参与测试。
     - 告知计划中的下一步改进方向和大致时间。

本指南可作为之后每次小范围测试的模板。若后续测试对象从“老师”扩展到“家长”，可以在此文档基础上增加“家长视角”的操作步骤和反馈要点。
